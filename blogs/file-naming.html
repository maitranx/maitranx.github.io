Naming Convention for file

  1. File Naming:

    File name format:

  2. Grouping pages:

    Pages are grouped following categories:

    • General pages: Cover, User flow cover
    • Documents: Design Navigation, Component Naming Convention
    • Features: A feature is a page. See more at 3.
    • Proposals: Design changes that are proposed and not approved by client yet. Similar to Branching in Figma
    • Others: depending on specific project

  3. Page Naming:

    Base on the Feature name to add a number. The page name will write follow the feature name.

    1. Page name format:

      1. Standard feature

        All order number should be start at 1 ==> example: 1. Swap

      2. Complex features

        In case, the feature is too complex, the feature will be broken into small pages in order to easily keep track.
        Press Cmd+Ctrl+Spacebar to insert the connector that represents the parent-child relationship of Parent Feature and Child Feature

    2. Page status format:

      🟨 Working in progress: Designers are working on it or the design are under discussion with product team
      ✅ Completed: Ready for Dev
      Note: Press Cmd+Ctrl+Spacebar to insert the symbol

  4. Organizing artboards

    1. Organizing flows and versions

      • Flow are organized vertically.
      • Mobile/Desktop and Light/Dark mode versions of a flow are organized horizontally.

    2. Flow Cover

    3. Updating feature

      1. Small change

        Regarding small changes, changes are made directly on existing flow and noted on cover.

      2. Big change

        Regarding big changes, designer follows step as below:
        Step 1: Name your original flows as Version 1.0

        Step 2: Duplicate only flows that have changes below and name them Version 1.1
        Step3: Remove/archive the original version of flows so that development team won’t be misunderstood

        Note: If there are small changes on Version 1.1, please follow our 4.3.1 Small change rule

    4. Flow Branching

      If a flow has more than one branch, 2nd cover will be added within the same flow to indicate different possible situations, leading to different sub-flow.

  5. Artboard Naming

    Each screen will have a number so team can easily communicate to each other on design

    Page name format:

    • 1st # Stand for feature number (also page number)
    • 2nd # Stand for flow number (*)
    • 3rd # Stand for screen number

    (*) The subsequent level will depend on the numerical value of the preceding level.
